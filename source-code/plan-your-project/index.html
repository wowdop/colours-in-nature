Therefore, we should split our projects into something like this:

  • Primary actions: where the main focus should be in the interface. For ex.: “log in”, “log out”, “submit”, “search”, “add to cart”, “cart”, “reviews”, etc.
  • Secondary actions: additional things linked to the primary actions. In general here there are actions that a user might want to do. For example: “more information”, “forgot your pw”, “suggestions”, “add”, “play x video”, “see x information”, “watch now”, “library”, etc.
  • Background: the background space, the space in between elements and needed (choosed) to wrap items in order to create breathing room and create a separation between categories of interest. The background provides contrast. The more elements need to be brought to the spotlight, the less similar they should be to the background. Even if it’s obvious, the closer the color is to the background, the more it blends into the background.
